# Jeremy's Communication Style Guide

This directory contains comprehensive documentation of Jeremy's (Jiki founder) communication patterns, teaching philosophy, and linguistic style. These guides help Claude generate content that authentically matches Jeremy's voice across different contexts.

## Source Material

All style documentation is derived from transcript analysis of:
- **JikiScript bootcamp live-stream** (variables.txt) - beginner audience, teaching session
- **Go course pre-production** (strings lesson) - experienced programmer audience, teaching session
- **Exercism Insiders update** (July 31, 2023) - community/supporter audience, business update

These three transcripts capture Jeremy's voice across different contexts: teaching beginners, teaching experienced developers, and speaking candidly to supporters about business challenges.

## Files in This Directory

### [writing.md](writing.md)

## Usage Guidance

**For blog posts:** Combine writing.md (structure/format) with speaking.md (linguistic patterns)
**For educational content:** Combine teaching.md with speaking.md
**For documentation:** Use lighter touch from principles.md
**For community updates:** Combine writing.md (structure) with speaking.md (emotional honesty)

## Key Takeaway

Jeremy's voice is defined by:
- **British English** patterns (whilst, quite, a bit, loads of)
- **Conversational hedging** (I think, maybe, probably, sort of)
- **Inclusive language** ("we" not "you")
- **Emotional honesty** (openly shares difficulty and limitations)
- **Simplicity advocacy** (actively filters complexity)
- **Concrete teaching** (physical metaphors, before/after framing)
